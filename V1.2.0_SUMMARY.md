### 2. 核心功能：支持编辑历史打卡 📝

**新增功能**：
- ✅ 可以选择历史日期
- ✅ 可以标记历史条目为"已打卡"或"未打卡（漏卡）"
- ✅ 保持数据一致性（快照机制）

**使用场景**：
```
场景1：补打卡
  - 昨天忘记打卡 → 今天补录

场景2：修正错误
  - 昨天误打卡 → 今天改为未打卡

场景3：批量补录
  - 周末漏打卡 → 周一一起补
```

**操作流程**：
```
1. 点击左箭头切换到历史日期
2. 点击条目
3. 选择"标记为已打卡"或"标记为未打卡（漏卡）"
4. 完成
```

**技术实现**：
- 新增云函数：`checkin-markHistorical`
- 修改前端逻辑：`pages/checkin/checkin.js`
- 数据一致性：快照机制 + 周期关联

**文件修改**：
- `cloudfunctions/checkin-markHistorical/` (新增)
- `miniprogram/pages/checkin/checkin.js` (修改)


**作用**：

- ✅ 不影响其他周期数据

## ⚠️ 注意事项

### 1. 修改历史会影响统计
```
标记历史打卡后：
- 本周累计会变化
- PK血条会变化
- 结算数据会变化
```

**建议**：
- 在本周内及时补录
- 避免跨周修改

---

### 2. 已结算周期
```
⚠️ 已结算的周期修改历史：
  - 需要重新结算
  - 或者接受数据不一致
```

**建议**：
- 只修改当前周期的历史
- 已结算周期谨慎修改

---

### 3. 数据完整性
```
✅ 快照机制保证：
  - 条目删除不影响历史
  - 条目修改不影响历史
  - 积分计算准确
```
