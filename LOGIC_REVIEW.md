## 🔴 发现的潜在问题

### 问题1：删除条目后的历史记录统计

**场景**：
1. 用户创建条目A（10分）
2. 本周打卡3次（30分）
3. 删除条目A
4. 查询本周积分 → **会不会找不到条目？**

**当前逻辑**：
```javascript
// settlement-getBalance/index.js
const itemsResult = await db.collection(COLLECTIONS.CHECKIN_ITEM).where({
  _id: _.in(itemIds)
}).get()

// 如果条目已删除，itemsResult 中找不到，积分计算会失败！
```

**风险等级**：🔴 高
**影响**：删除条目会导致历史积分丢失

**解决方案**：
1. 打卡记录中保存 `points` 快照
2. 或者软删除条目（不物理删除）

---

### 问题2：禁用条目是否影响漏卡生成

**场景**：
1. 用户创建条目A（启用）
2. 23:50 禁用条目A
3. 23:59 定时任务检查 → **会生成漏卡吗？**

**当前逻辑**：
```javascript
// timer-dailyCheck/index.js
const items = await db.collection(COLLECTIONS.CHECKIN_ITEM).where({
  userId: user._id,
  enabled: true  // ✅ 只检查启用的条目
}).get()
```

**结论**：✅ 不会生成漏卡（逻辑正确）

---

### 问题3：周期结算失败的容错

**场景**：
1. 周日23:59 系统故障
2. 结算未执行
3. 周一00:00 用户打卡 → **会关联到哪个周期？**

**当前逻辑**：
```javascript
// timer-weeklySettlement/index.js
// 1. 结算本周
// 2. 创建下周周期
// 3. 更新用户 currentPeriodId

// 如果第1步失败，后续都不执行
```

**风险等级**：🟡 中
**影响**：用户打卡会失败（没有周期）

**解决方案**：
1. 登录时检查：如果没有周期，自动创建
2. 打卡时检查：如果周期无效，自动修复

---

const missedDiff = partnerMissed - myMissed

if (missedDiff !== 0) {

## 🔧 核心问题修复

### 🔴 必须修复：问题1 - 删除条目导致积分丢失

**方案A：打卡记录保存积分快照**（推荐）

修改打卡记录结构：
```javascript
{
  userId: xxx,
  itemId: xxx,
  itemTitle: "晨跑",     // ✅ 新增：条目标题快照
  itemPoints: 10,        // ✅ 新增：条目积分快照
  periodId: xxx,
  date: "2026-03-23",
  status: "completed",
  createdAt: Date
}
```

**优点**：
- 删除条目不影响历史数据
- 统计速度快（不需要join）
- 历史记录完整可追溯

**缺点**：
- 数据冗余
- 需要修改现有逻辑

---

**方案B：软删除条目**

修改删除逻辑：
```javascript
// 不物理删除，只标记
{
  enabled: false,
  deleted: true,       // ✅ 新增字段
  deletedAt: Date
}
```

**优点**：
- 改动小
- 历史数据不受影响

**缺点**：
- 条目列表需要过滤 deleted
- 数据量持续增长

---

### 🟡 建议修复：问题3 - 周期结算容错

**修改登录逻辑**：
```javascript
// user-login/index.js
if (!user.currentPeriodId) {
  // ✅ 自动创建周期
  const periodId = await createPeriod(user._id)
  user.currentPeriodId = periodId
}
```

**修改打卡逻辑**：
```javascript
// checkin-submit/index.js
if (!user.currentPeriodId) {
  throw new Error('没有活跃周期，请联系管理员')
}

// ✅ 验证周期有效性
const period = await db.collection(COLLECTIONS.PERIOD).doc(user.currentPeriodId).get()
if (!period.data || period.data.status !== 'active') {
  // 自动修复
  const periodId = await createPeriod(user._id)
  user.currentPeriodId = periodId
}
```
