## 🧪 阶段2: 体验版（推荐先做这个）

### 为什么需要体验版？

在提交正式版审核之前，用体验版测试：
1. ✅ 在真实手机上测试功能
2. ✅ 邀请伙伴（女朋友）一起测试PK功能
3. ✅ 发现并修复bug
4. ✅ 避免正式版审核被拒

### 如何生成体验版？

#### 步骤1: 登录小程序后台

```
浏览器打开：https://mp.weixin.qq.com/
  ↓
用管理员微信扫码登录
  ↓
进入「版本管理」
```

#### 步骤2: 设为体验版

```
版本管理 → 开发版本
  ↓
找到刚刚上传的版本（显示版本号和时间）
  ↓
点击「选为体验版」按钮
  ↓
确认
```

#### 步骤3: 添加体验成员

```
成员管理 → 体验成员
  ↓
点击「添加成员」
  ↓
输入微信号或扫码邀请
  ↓
最多可添加15个体验成员
```

**推荐添加**：
- ✅ 你自己的微信
- ✅ 你女朋友的微信
- ✅ 测试账号

#### 步骤4: 扫码体验

```
版本管理 → 体验版本
  ↓
显示体验版二维码
  ↓
手机微信扫码
  ↓
进入小程序体验
```

### 体验版的特点：

| 特性 | 说明 |
|------|------|
| **无需审核** | 设为体验版后立即可用 |
| **真实环境** | 使用正式的云环境和数据库 |
| **限制人数** | 最多15个体验成员 |
| **功能完整** | 和正式版功能一样 |
| **可以更新** | 随时上传新版本替换 |

## 🎉 阶段3: 正式版（最终发布）

### 何时提交正式版？

**建议**：体验版测试1-2天后，确认没有问题再提交

**测试清单**：
- [ ] 用户登录和绑定伙伴
- [ ] 创建打卡条目
- [ ] 完成打卡
- [ ] 取消打卡
- [ ] PK页面数据正确
- [ ] 查看历史打卡记录
- [ ] 周期自动切换（等到下周一）
- [ ] 结算功能

### 提交审核步骤：

#### 步骤1: 在小程序后台提交审核

```
浏览器打开：https://mp.weixin.qq.com/
  ↓
版本管理 → 开发版本
  ↓
找到要发布的版本
  ↓
点击「提交审核」按钮
```

#### 步骤2: 填写审核信息

**必填信息**：

1. **小程序介绍**
   ```
   示例：Daily Duel是一款情侣打卡PK小程序，
   通过每日打卡和周结算机制，帮助情侣们互相监督，
   共同完成目标，增进感情。
   ```

2. **服务类目**
   ```
   建议选择：
   - 生活服务 > 综合生活服务
   或
   - 教育 > 在线教育
   ```

3. **用户隐私保护指引**
   ```
   说明收集的用户信息：
   - 微信昵名、头像（用于显示用户信息）
   - 打卡记录（用于PK功能）
   ```

4. **测试账号（重要！）**
   ```
   提供2个测试账号（你和伙伴的微信号）
   审核人员会用这些账号测试功能
   
   账号1（你）：微信号 + 密码（无密码可留空）
   账号2（伙伴）：微信号 + 密码
   
   说明：两个账号已互相绑定为伙伴
   ```

5. **功能页面截图**
   ```
   上传主要功能页面的截图：
   - 首页
   - 今日打卡页面
   - PK对战页面
   - 条目管理页面
   - 结算页面
   ```

#### 步骤3: 等待审核

```
审核时间：通常1-7天
  ↓
可能结果：
  • 审核通过 → 可以发布
  • 审核被拒 → 查看拒绝原因 → 修改 → 重新提交
```

**常见拒绝原因**：
1. ❌ 服务类目不符
2. ❌ 测试账号无法登录
3. ❌ 功能不完整或有bug
4. ❌ 缺少隐私政策
5. ❌ 存在支付功能但未申请支付权限

#### 步骤4: 审核通过后发布

```
收到审核通过通知
  ↓
版本管理 → 审核版本
  ↓
点击「发布」按钮
  ↓
确认发布
  ↓
正式版上线！
```

#### 步骤5: 用户如何找到你的小程序

**方式1: 搜索**
```
微信 → 发现 → 小程序 → 搜索「Daily Duel」
```

**方式2: 分享**
```
你的小程序 → 右上角 ··· → 转发给朋友
```

**方式3: 小程序码**
```
小程序后台 → 设置 → 基本设置 → 小程序码
  → 下载小程序码
  → 分享给别人扫码
```

## 🔄 后续更新流程

发布正式版后，如果要更新：

```
1. 修改代码
   ↓
2. 点击「上传」（上传新版本）
   ↓
3. 设为体验版 → 测试
   ↓
4. 提交审核
   ↓
5. 审核通过 → 发布
   ↓
6. 用户下次打开小程序时自动更新
```

**注意**：
- ⚠️ 每次修改前端代码都需要重新审核
- ⚠️ 只修改云函数不需要审核（直接上传云函数即可）

## 📞 遇到问题？

### 审核被拒怎么办？

1. **查看拒绝原因**
   ```
   版本管理 → 审核版本 → 查看拒绝原因
   ```

2. **根据原因修改**
   - 服务类目问题 → 重新选择类目
   - 功能问题 → 修改代码
   - 测试账号问题 → 提供可用的测试账号

3. **重新提交**
   ```
   修改后 → 上传新版本 → 再次提交审核
   ```

### 如何联系微信审核团队？

```
小程序后台 → 帮助与反馈 → 提交工单
```
