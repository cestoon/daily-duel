# 部署指南

## 前置要求

1. 注册微信小程序账号
2. 开通云开发服务
3. 获取 AppID 和环境ID

## 部署步骤

### 第一步：配置项目

1. 修改 `project.config.json` 中的 `appid` 为你的小程序 AppID
2. 修改 `miniprogram/app.js` 中的 `env` 为你的云开发环境ID

```json
// project.config.json
{
  "appid": "your-appid-here",
  ...
}

// miniprogram/app.js
wx.cloud.init({
  env: 'your-env-id',
  traceUser: true
})
```

### 第二步：打开项目

1. 打开微信开发者工具
2. 选择"小程序" -> "导入项目"
3. 选择项目目录并点击"导入"
4. 项目结构如下：
   - `miniprogram/` - 小程序前端代码
   - `cloudfunctions/` - 云函数代码
   - `project.config.json` - 项目配置

### 第三步：初始化云开发

1. 在微信开发者工具中，点击顶部菜单"云开发"
2. 如果还没有云开发环境，点击"开通"
3. 选择或创建环境，记录环境ID

### 第四步：创建数据库集合

1. 在云开发控制台中，进入"数据库"标签
2. 创建以下集合（Collection）：
   - `User` - 用户表
   - `Period` - 周期表
   - `CheckinItem` - 打卡条目表
   - `CheckinRecord` - 打卡记录表
   - `Settlement` - 结算表
   - `ReminderConfig` - 提醒配置表

### 第五步：部署云函数

1. 在云函数目录中，右键点击每个云函数文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待部署完成

按以下顺序部署云函数：
1. `init-db` - 数据库初始化
2. `user/login` - 用户登录
3. `user/getInfo` - 获取用户信息
4. `user/getPartner` - 获取伙伴信息
5. `user/bindPartner` - 绑定伙伴
6. `period/create` - 创建周期
7. `checkin/submit` - 提交打卡
8. `checkin/getItems` - 获取条目列表
9. `checkin/getTodayRecords` - 获取今日记录
10. `checkin/addItem` - 添加条目
11. `checkin/updateItem` - 更新条目
12. `checkin/deleteItem` - 删除条目
13. `settlement/getBalance` - 获取余额
14. `settlement/confirmPayment` - 确认支付
15. `settlement/getList` - 获取结算列表
16. `timer/dailyCheck` - 每日打卡检查
17. `timer/weeklySettlement` - 每周结算

### 第六步：运行数据库初始化

1. 在云开发控制台中，找到 `init-db` 云函数
2. 点击"测试"
3. 点击"调用"
4. 查看返回结果，确认数据库初始化成功

### 第七步：配置定时触发器

定时触发器配置已包含在云函数的 `config.json` 文件中，部署云函数时会自动创建。

### 第八步：配置订阅消息（可选）

1. 在微信公众平台中申请订阅消息模板
2. 在云函数中配置模板ID
3. 在需要发送消息的地方调用发送订阅消息的API

## 测试部署

### 1. 测试登录

1. 在小程序中点击"微信登录"
2. 确认能够正常登录并获取用户信息

### 2. 测试绑定伙伴

1. 在设置页面找到用户ID
2. 使用另一个账号登录
3. 输入对方的用户ID进行绑定
4. 确认绑定成功

### 3. 测试打卡功能

1. 添加打卡条目
2. 在今日打卡页面进行打卡
3. 确认打卡记录正常保存

### 4. 测试结算功能

1. 等待每周一自动结算
2. 或手动触发 `timer/weeklySettlement` 云函数
3. 确认结算记录创建成功

## 常见问题

### 1. 云函数部署失败

- 检查是否有足够的云函数配额
- 检查依赖包是否正确安装
- 查看云函数日志定位错误

### 2. 数据库操作失败

- 检查数据库集合是否正确创建
- 检查数据库权限设置
- 查看云函数日志定位错误

### 3. 定时任务不执行

- 检查定时触发器配置是否正确
- 检查云函数是否正常部署
- 查看定时任务执行日志

### 4. 订阅消息发送失败

- 检查用户是否授权订阅消息
- 检查模板ID是否正确
- 检查参数格式是否符合要求

## 监控和维护

### 查看日志

1. 在云开发控制台中，进入"云函数"标签
2. 点击云函数名称
3. 查看"日志"标签页

### 查看数据

1. 在云开发控制台中，进入"数据库"标签
2. 选择集合查看数据
3. 可以进行增删改查操作

### 监控定时任务

1. 在云开发控制台中，进入"云函数"标签
2. 点击云函数名称
3. 查看"定时触发器"标签页

## 性能优化

1. **数据库索引**: 已在 `init-db` 中创建必要索引
2. **云函数并发**: 根据实际使用情况调整并发配置
3. **缓存策略**: 小程序端使用本地缓存减少云函数调用

## 安全建议

1. **权限控制**: 设置合适的数据库权限
2. **数据验证**: 在云函数中验证用户输入
3. **日志记录**: 记录关键操作日志便于追踪
