## 🔧 上线步骤

### 第1步：配置定时触发器（5分钟）⏰

**必须配置！否则漏卡检查和周结算不会自动执行！**

#### 1.1 配置每日漏卡检查

```
腾讯云微信云开发控制台
→ 云函数
→ timer-dailyCheck
→ 触发器
→ 创建触发器

配置：
- 触发器名称: daily-check
- 触发类型: 定时触发器
- Cron 表达式: 0 59 23 * * * *
  （解释：每天 23:59:00 执行）
- 备注: 每日检查漏卡
→ 确定
```

#### 1.2 配置每周自动结算

```
腾讯云微信云开发控制台
→ 云函数
→ timer-weeklySettlement
→ 触发器
→ 创建触发器

配置：
- 触发器名称: weekly-settlement
- 触发类型: 定时触发器
- Cron 表达式: 0 59 23 * * 0
  (解释：每周日 23:59:00 执行)
- 备注: 每周自动结算
→ 确定
```

#### 1.3 验证触发器

```
云函数 → timer-dailyCheck → 触发器
→ 查看触发器列表
→ 确认 daily-check 已创建 ✅

云函数 → timer-weeklySettlement → 触发器
→ 查看触发器列表
→ 确认 weekly-settlement 已创建 ✅
```

**重要提示**：
- ⚠️ Cron 表达式格式：`秒 分 时 日 月 周`
- ⚠️ 时区：默认为 UTC+8（北京时间）
- ⚠️ 首次配置后，需等到指定时间才会执行

## 🐛 常见问题处理

### Q1: 定时任务没有执行？

**检查**：
```
1. 触发器是否配置正确
2. Cron 表达式是否正确
3. 云函数是否有报错
4. 查看云函数日志
```

**解决**：
```
→ 重新配置触发器
→ 手动触发测试
→ 检查代码逻辑
```

---

### Q2: 用户无法绑定伙伴？

**检查**：
```
1. 邀请码是否正确
2. 是否已绑定其他伙伴
3. 云函数是否报错
```

**解决**：
```
→ 查看用户数据
→ 清除错误绑定
→ 重新生成邀请码
```

---

### Q3: 打卡数据不显示？

**检查**：
```
1. 数据库记录是否存在
2. 日期格式是否正确
3. 快照字段是否完整
```

**解决**：
```
→ 查看 checkin_record 集合
→ 检查记录 status 和 date
→ 验证快照字段
```

---

### Q4: 结算金额计算错误？

**检查**：
```
1. 漏卡记录是否完整
2. 积分快照是否正确
3. 结算逻辑是否执行
```

**解决**：
```
→ 运行 test-runner 测试
→ 查看结算记录
→ 手动触发 weeklySettlement
```
