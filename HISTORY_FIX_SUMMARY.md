# 🔧 历史日期显示修复总结

**问题发现**：2026-03-24
**修复版本**：v1.1.1

---

## 🐛 问题描述

**现象**：
- 23号有多个条目，但查看23号历史时只显示2个
- 今天（24号）的条目数量正常

**用户疑问**：
> "我并没有增减条目，为什么23号历史显示的条目少了？"

---

## 🔍 问题根因

### 原始逻辑（错误）

```javascript
历史日期显示流程：
1. 查询当前启用的条目列表（enabled: true）
2. 过滤出在该日期之前创建的条目
3. 查询该日期的打卡记录
4. 匹配显示
```

**问题**：
- ❌ 只查询 `enabled: true` 的条目
- ❌ 如果23号有条目，但24号被禁用 → 不显示
- ❌ 如果23号有条目，但后来删除 → 不显示
- ❌ 无法准确还原23号的真实打卡情况

**实际场景**：
```
23号：有4个条目（运动、学习、阅读、健身）
      - 其中2个打卡，2个漏卡
      
24号：用户禁用了"阅读"和"健身"

查看23号历史：
→ 只查询 enabled=true 的条目
→ 只返回"运动"和"学习"
→ 显示2个 ❌（应该显示4个）
```

---

## ✅ 修复方案

### 新逻辑：从打卡记录反推

```javascript
历史日期显示流程：
1. 直接查询该日期的所有打卡记录（completed + missed）
2. 从记录中提取条目信息（使用快照字段）
3. 显示
```

**优点**：
- ✅ 完全准确（还原当天真实情况）
- ✅ 不受后续条目变更影响（禁用、删除、修改）
- ✅ 充分利用快照机制（itemTitle, itemPoints）
- ✅ 已删除的条目也能正常显示

**数据来源**：
- `itemId`: 记录中的 itemId
- `title`: 记录中的 **itemTitle**（快照）
- `points`: 记录中的 **itemPoints**（快照）
- `status`: 记录中的 status（completed/missed）

**修改后**：
```javascript
async loadData() {
  if (this.data.isToday) {
    // 今日：查询启用条目 + 打卡记录（原逻辑）
    const itemsRes = await wx.cloud.callFunction({
      name: 'checkin-getItems'
    })
    // ...
    
  } else {
    // ✅ 历史日期：从打卡记录反推
    const recordsRes = await wx.cloud.callFunction({
      name: 'checkin-getRecordsByDate',
      data: { date: this.data.currentDate }
    })
    
    // 从记录构建条目列表（使用快照）
    const items = records.map(record => ({
      _id: record.itemId,
      title: record.itemTitle || '已删除的条目',
      points: record.itemPoints || 10,
      checked: record.status === 'completed',
      status: record.status,
      statusText: record.status === 'completed' ? '已完成' : '漏卡',
      isHistorical: true
    }))
    
    this.setData({ items })
  }
}
```

## ⚠️ 注意事项

### 1. 依赖快照字段

**要求**：
- 打卡记录必须包含 `itemTitle` 和 `itemPoints` 快照
- 这是 v1.1.0 新增的功能

**兼容性**：
```javascript
title: record.itemTitle || '已删除的条目',
points: record.itemPoints || 10
```
- 如果快照不存在（旧数据），使用默认值
- 不会报错，但显示可能不准确

---

### 2. 只显示有记录的条目

**限制**：
- 只显示 completed 或 missed 的条目
- 如果某个条目既没打卡也没被标记漏卡，不显示

**原因**：
- 定时任务 `timer-dailyCheck` 会在23:59标记漏卡
- 正常情况下，所有启用的条目都会有记录

**极端情况**：
- 如果定时任务失败，某些条目可能没有记录
- 这种情况下，历史查询不会显示这些条目
- **但这本身就是数据异常，应该修复定时任务**
