# 📝 历史打卡编辑功能

**版本**：v1.2.0
**更新时间**：2026-03-25

---

## 🎯 功能说明

### 新增功能

**支持编辑历史日期的打卡记录**：
- ✅ 可以选择过去的日期
- ✅ 可以标记为"已打卡"或"未打卡（漏卡）"
- ✅ 保持数据一致性
- ✅ 快照机制保证历史数据准确

---

## 📋 使用场景

### 场景 1：补打卡
```
情况：昨天忘记打卡了
操作：
1. 切换到昨天的日期
2. 点击条目
3. 选择"标记为已打卡"
4. 确认
```

---

### 场景 2：修正错误
```
情况：昨天误打卡了，实际没做
操作：
1. 切换到昨天的日期
2. 点击条目
3. 选择"标记为未打卡（漏卡）"
4. 确认
```

---

### 场景 3：批量补录
```
情况：周末忘记打卡，周一一起补
操作：
1. 切换到周六
2. 标记各条目状态
3. 切换到周日
4. 标记各条目状态
```

## 🔧 技术实现

### 前端逻辑

#### 1. 日期切换
```javascript
// 今日 vs 历史日期
if (this.data.isToday) {
  // 今日：查询启用条目 + 今日记录
  // 可以正常打卡/取消打卡
} else {
  // 历史：查询所有条目 + 该日期记录
  // 点击显示操作选项
}
```

---

#### 2. 操作交互
```javascript
// 历史日期点击条目
wx.showActionSheet({
  itemList: [
    '标记为已打卡',
    '标记为未打卡（漏卡）'
  ]
})
```

---

#### 3. 数据加载
```javascript
// 历史日期数据加载流程
1. 获取所有启用的条目
2. 获取该日期的打卡记录
3. 合并：有记录则显示状态，无记录则显示"未打卡"
```

---

### 后端逻辑

#### 云函数：checkin-markHistorical

**功能**：标记历史打卡记录

**参数**：
```javascript
{
  itemId: string,    // 条目ID
  date: string,      // 日期 'YYYY-MM-DD'
  status: string     // 'completed' 或 'missed'
}
```

**逻辑**：
```
1. 参数验证
   - 检查必需参数
   - 验证状态值
   - 验证日期（不能是未来）

2. 查询用户和条目
   - 获取用户ID
   - 获取条目信息（用于快照）

3. 检查是否已有记录
   - 有记录：更新状态 + 快照
   - 无记录：创建新记录 + 快照

4. 返回结果
   - action: 'update' 或 'create'
```

## 🛡️ 数据一致性保证

### 1. 快照机制
```javascript
// 保存条目快照到打卡记录
{
  itemTitle: item.title,    // 条目标题快照
  itemPoints: item.points   // 条目积分快照
}
```

**作用**：
- ✅ 条目删除后，历史记录仍能显示
- ✅ 条目修改后，历史记录不受影响
- ✅ 结算计算准确

---

### 2. 状态约束
```javascript
// 历史日期只有两种状态
status: 'completed' | 'missed'

// 今日有三种状态
status: 'completed' | 'missed' | 'pending'
```

---

### 3. 日期验证
```javascript
// 不能标记未来日期
if (targetDate > today) {
  return { success: false, message: '不能标记未来日期' }
}
```

---

### 4. 周期关联
```javascript
// 记录关联到当前周期
periodId: user.currentPeriodId
```

**作用**：
- ✅ 结算时统计该周期的所有记录
- ✅ 不会影响其他周期的数据

## ⚠️ 注意事项

### 1. 不能标记未来
```
❌ 不能标记明天的打卡
✅ 只能标记今天和过去
```

---

### 2. 修改历史会影响统计
```
⚠️ 标记历史打卡后：
  - 本周累计会变化
  - PK血条会变化
  - 结算数据会变化
```

---

### 3. 已结算周期不建议修改
```
⚠️ 如果修改了已结算的日期：
  - 需要重新结算（手动）
  - 或者接受数据不一致
```

**建议**：
- 只允许修改当前周期的历史日期
- 已结算周期的历史日期标记为只读
