## 🎯 测试目标

验证首页整改是否成功，确保符合审核要求。

---

## 🚀 快速测试（5分钟）

### Step 1: 清除缓存

```
微信开发者工具
→ 工具栏
→ 清除缓存
→ 清除所有数据存储（Storage）
→ 确认
```

**目的**：模拟未登录用户

---

### Step 2: 重新编译

```
点击"编译"按钮
或
快捷键：Cmd + B (Mac) / Ctrl + B (Windows)
```

---

### Step 3: 查看首页

**应该看到**：
- ✅ Logo 和 "Daily Duel" 标题
- ✅ 标语 "情侣打卡PK，一起变更好"
- ✅ 4个功能介绍卡片：
  - 📝 每日打卡
  - ⚔️ PK对战
  - 🏆 周结算
  - 📊 数据统计
- ✅ "开始使用" 按钮
- ✅ 底部提示文字

**不应该看到**：
- ❌ "微信登录" 按钮（直接显示）
- ❌ PK页面
- ❌ 打卡页面

---

### Step 4: 测试登录弹窗

```
1. 点击"开始使用"按钮
2. 应该弹出登录弹窗 ✅
3. 查看弹窗内容：
   - "授权登录" 标题
   - "✕" 关闭按钮（右上角）
   - Logo图标
   - "需要获取您的昵称和头像"
   - "用于完善个人资料和伙伴识别"
   - "取消" 按钮
   - "授权登录" 按钮
```

---

### Step 5: 测试取消功能

**测试1：点击"取消"按钮**
```
1. 点击弹窗底部"取消"按钮
2. 弹窗应该关闭 ✅
3. 停留在介绍页 ✅
```

**测试2：点击"✕"按钮**
```
1. 再次点击"开始使用"
2. 点击弹窗右上角"✕"按钮
3. 弹窗应该关闭 ✅
4. 停留在介绍页 ✅
```

**测试3：点击背景**
```
1. 再次点击"开始使用"
2. 点击弹窗外的灰色背景
3. 弹窗应该关闭 ✅
4. 停留在介绍页 ✅
```

---

### Step 6: 测试登录功能

```
1. 点击"开始使用"
2. 点击"授权登录"
3. 在弹出的授权框点击"允许" ✅
4. 显示"登录成功" ✅
5. 1.5秒后自动跳转到PK页 ✅
```

---

### Step 7: 测试已登录状态

```
1. 已经登录成功
2. 关闭开发者工具的小程序窗口
3. 重新点击"编译"
4. 应该直接打开PK页（不显示介绍页）✅
```

## 🐛 常见问题

### Q1: 重新编译还是看不到首页

**A**: 清除缓存

```
微信开发者工具
→ 清除缓存
→ 清除所有数据存储
→ 重新编译
```

---

### Q2: 登录后没有跳转

**A**: 检查代码是否保存

```
1. 确认 index.js 已保存
2. 重新编译
3. 清除缓存后测试
```

---

### Q3: 真机看不到首页

**A**: 重新上传预览

```
1. 重新点击"预览"
2. 生成新的二维码
3. 删除手机上的小程序
4. 重新扫码
```

---

### Q4: 弹窗关闭不了

**A**: 检查事件绑定

```
1. 确认 catchtap 事件正确
2. 重新编译
```
