## ⚠️ 注意事项

### 1. 定时触发器配置

**周结算任务** (`timer-weeklySettlement`) 的触发器配置：

```
名称: weekly-settlement
类型: 定时触发器
Cron表达式: 0 0 10 * * 1 *
描述: 每周一00:10执行（结算上周+创建新周期）
```

**验证方法**：

在云开发控制台：
- 云函数 → `timer-weeklySettlement` → 触发器
- 检查是否存在定时触发器
- 如果没有，手动添加

### 2. 临时云函数清理

测试完成后，可以删除以下临时云函数：
- `scripts-diagnose`
- `scripts-forceNewPeriod`

**或者保留它们**用于将来的数据维护。

### 3. 数据备份

在生产环境部署前，建议备份数据库：

1. 云开发 → 数据库
2. 导出以下集合：
   - User
   - Period
   - CheckinRecord
   - CheckinItem

## 🚨 常见问题

### Q1: 部署后PK页面还是显示旧数据

**A**: 清除缓存并重新登录：
1. 开发者工具 → 清缓存 → 全部清除
2. 模拟器 → 清除数据缓存
3. 重新编译小程序

### Q2: 诊断工具显示"没有活动周期"

**A**: 运行修复工具 `scripts-forceNewPeriod`

### Q3: 修复工具运行失败

**A**: 检查云函数日志，常见原因：
- 云函数未正确上传
- package.json 缺失
- 数据库权限问题

查看日志：云开发 → 云函数 → 选择云函数 → 日志

### Q4: 本周数据包含上周的记录

**A**: 运行修复工具后，上周的记录仍然存在，但：
- 它们的 `periodId` 指向旧周期
- 统计时会被过滤掉（只统计当前周期）
- 如需清理，手动删除或让它们保留2年后自动清理
